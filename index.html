Introduction

This page covers the key elements of HTML and HTML5 using pure markup. Each section lists core tags with short explanations.

Document Structure

Note: Basic layout of an HTML document.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Title</title>
  </head>
  <body>
    <!-- content -->
  </body>
</html>

Metadata

Note: Elements in the <head>.

  • <meta charset="UTF-8"> – character encoding.
  • <meta name="viewport" content="width=device-width, initial-scale=1"> – responsive design.
  • <title> – document title.
  • <link rel="icon" href="favicon.ico"> – favicon.
  • <link> – external resource link (e.g., CSS).
  • <style> – internal CSS.
  • <base> – base URL for relative links.
<head>
  <meta charset="UTF-8">
  <title>My Page</title>
  <link rel="icon" href="favicon.ico">
  <link rel="stylesheet" href="style.css">
  <style>
    body { font-family: Arial, sans-serif; }
  </style>
</head>

Text Basics

Note: Headings and paragraphs.

<h1>Main Heading</h1>
<h2>Subheading</h2>
<p>Paragraph text.</p>

Inline Text Semantics

Note: Mark up words or phrases.

  • <em> – emphasis
  • <strong> – strong importance
  • <small> – small print
  • <mark> – highlight
  • <code> – inline code
  • <abbr title="abbreviation">abbr</abbr> – abbreviation
  • <time datetime="2024-01-01"> – date or time
<p>
  Use <em>emphasis</em> and <strong>strong</strong> text,
  <code>code</code> and <mark>highlight</mark>.
</p>

Grouping Content

Note: Block-level elements.

  • <div> – generic container.
  • <p> – paragraph.
  • <hr> – thematic break.
  • <blockquote> – quoted section.
  • <pre> – preformatted text.
<div>
  <blockquote>Quote</blockquote>
  <hr />
</div>

Forms

Note: Collecting user input.

<form action="/submit" method="post">
  <fieldset>
    <legend>Contact</legend>
    <label for="name">Name</label>
    <input type="text" id="name" name="name" required>
    <input type="checkbox" id="subscribe" name="subscribe"> Subscribe
    <input type="file" name="file">
    <textarea name="message"></textarea>
    <select name="options">
      <option value="one">One</option>
    </select>
    <input type="submit" value="Send">
  </fieldset>
</form>
  • Common controls: <input>, <textarea>, <select>, <button>, <fieldset>, <legend>, <label>

Accessibility

Note: ARIA and best practices.

  • Use semantic elements for structure.
  • alt text on images.
  • ARIA roles: role="navigation", etc.
  • Label form controls with <label>.
  • Use aria-label, aria-labelledby, aria-describedby for screen readers.
  • Keyboard navigation: tabindex, accesskey.
  • Ensure color contrast and visible focus states.
<nav role="navigation" aria-label="Main menu">Menu</nav>
<form>
  <label for="email">Email</label>
  <input id="email" type="email" aria-describedby="emailHelp">
  <small id="emailHelp">We'll never share your email.</small>
</form>

Best Practices

Note: General tips.

  • Keep markup clean and well-indented.
  • Use semantic elements for clarity.
  • Include lang attribute on <html>.
  • Avoid deprecated tags like <font>.
<html lang="en">
  <body>
    <p>Clean code</p>
  </body>
</html>

Deprecated Tags

  • <font> (use CSS for styling)
  • <center> (use text-align: center; in CSS)
  • <bgsound> (use <audio> instead)
